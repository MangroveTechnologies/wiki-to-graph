# Outputs & example workflows

This repo ships a complete, runnable example: the wiki in
[`examples/llm-wiki/`](../examples/llm-wiki) is the exact wiki the committed
`build/` artifacts were generated from. Everything below runs from the repo
root against that example — no external files needed.

`examples/llm-wiki/` is a 28-page LLM wiki (the ["LLM wiki" pattern by Andrej
Karpathy](https://datasciencedojo.com/blog/llm-wiki-tutorial/)) compiled from 6
foundational AI papers:

- `wiki/` — 26 single-concept entity pages + `index.md` and `log.md` hub pages,
  linked with `[[wiki-links]]`.
- `raw/` — the 6 source-paper markdowns the entity pages cite in their
  `## Sources` sections (Attention Is All You Need, BERT, GPT-3, Foundation
  Models, RLHF/InstructGPT, Chinchilla).

---

## What each `build/` object is

Running `build` (below) regenerates every file in `build/`. They are all
**derived** from `examples/llm-wiki/wiki` — the markdown is the source of truth.

| File | Format | What it's for |
|------|--------|---------------|
| **`graph.json`** | JSON (node-link) | **Canonical output.** Loads directly into NetworkX / D3 / Cytoscape, is diffable in git, and is the input to every other subcommand (`validate`, `analyze`, `query`) and to the viewer. Preserves parallel typed edges (`multigraph: true`). |
| **`graph.db`** | SQLite | A relational view of the same graph (`nodes` + `edges` tables). Query it with plain SQL, or do recursive-CTE traversals. Binary — regenerated by `--emit sqlite`. |
| **`graph.graphml`** | GraphML (XML) | Import into graph GUIs like **Gephi** or **yEd** for layout/visual analysis. Regenerated by `--emit graphml`. |
| **`domain.json`** | JSON (KST) | Optional **Knowledge Space Theory** projection (`items` = concepts, `prerequisites` = the heuristic `--dag-edges` subset). Regenerated by `--kst`. The prerequisites are candidate suggestions from body references — curate before treating them as a true KST surmise relation. |
| **`graph-viewer.html`** | HTML | A self-contained, **fully offline** interactive viewer (no CDN, no network). Nodes colored by kind, edges by type; click a node to read its summary and walk its edges. Generated by `build_graph_viewer.py`. |

## Example workflows

`SCR=skills/wiki-to-graph/scripts`

### A · Full pipeline from the example wiki (reproduces `build/`)

```bash
SCR=skills/wiki-to-graph/scripts

# 1) BUILD every output from the example wiki
python3 $SCR/wiki_to_graph.py build examples/llm-wiki/wiki \
        -o build/graph.json --emit sqlite,graphml --kst

# 2) VALIDATE (exit 1 on dangling links / orphans / self-loops)
python3 $SCR/wiki_to_graph.py validate build/graph.json

# 3) ANALYZE (PageRank, in-degree, contested nodes, components, communities)
python3 $SCR/wiki_to_graph.py analyze build/graph.json --top 5

# 4) VIEW (offline interactive HTML)
python3 $SCR/build_graph_viewer.py build/graph.json -o build/graph-viewer.html
# then open build/graph-viewer.html in a browser
```

### B · Ask the graph questions (no SQL / no graph code)

```bash
python3 $SCR/wiki_to_graph.py query build/graph.json node "RLHF"            # details + edges
python3 $SCR/wiki_to_graph.py query build/graph.json neighbors "GPT-3"      # outgoing
python3 $SCR/wiki_to_graph.py query build/graph.json backlinks "Transformer" # incoming
python3 $SCR/wiki_to_graph.py query build/graph.json contradicts            # all tension pairs
```

### C · Filtered traversals

```bash
# Walk only 'related' edges out of Transformer
python3 $SCR/wiki_to_graph.py query build/graph.json bfs "Transformer" --edges related

# Walk the contradiction graph around GPT-3, treating edges as undirected
python3 $SCR/wiki_to_graph.py query build/graph.json dfs "GPT-3" --edges contradicts --undirected

# Shortest path between two concepts
python3 $SCR/wiki_to_graph.py query build/graph.json path "Positional Encoding" "RLHF"
```

### D · Edit the source, then rebuild (the graph is derived — never hand-edit `graph.json`)

```bash
python3 $SCR/wiki_to_graph.py update examples/llm-wiki/wiki \
        add-node --title "Mixture of Experts" --kind schema --summary "..."
python3 $SCR/wiki_to_graph.py update examples/llm-wiki/wiki \
        add-edge --from "Mixture of Experts" --to "Transformer" --type related

# re-run build to regenerate every output
python3 $SCR/wiki_to_graph.py build examples/llm-wiki/wiki -o build/graph.json --emit sqlite,graphml --kst
```

### E · Load `graph.json` into NetworkX for heavier analysis

```python
import json, networkx as nx
from networkx.readwrite import json_graph

G = json_graph.node_link_graph(json.load(open("build/graph.json")), edges="links")  # MultiDiGraph
concepts = [n for n, d in G.nodes(data=True) if d.get("type") == "concept"]
print(nx.pagerank(G.subgraph(concepts)))
```

### F · Query the SQLite view directly

```bash
sqlite3 build/graph.db "SELECT id, kind FROM nodes WHERE type='concept' ORDER BY id LIMIT 10;"
```
